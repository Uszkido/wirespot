# Code of Conduct

WireSpot is maintained for professional hotspot operators, network engineers,
and Vexel Innovations collaborators. We expect every contributor to keep the
project respectful, secure, and practical.

## Expected Behavior

- Be respectful and constructive.
- Keep discussions focused on WireSpot, Flutter, Android, RouterOS, WireGuard,
  printing, reporting, and operator workflows.
- Report bugs with clear reproduction steps.
- Protect customer, router, VPN, and credential information.
- Credit original work and avoid copying proprietary products.

## Unacceptable Behavior

- Harassment, insults, threats, or discriminatory language.
- Sharing real router passwords, VPN private keys, customer data, or private
  business records in issues, commits, screenshots, or logs.
- Uploading copied proprietary code or assets.
- Encouraging insecure deployment, such as exposing RouterOS API ports publicly.
- Spam, unrelated promotion, or disruptive behavior.

## Security-Sensitive Discussions

If a report involves credentials, bypasses, VPN configuration, authentication,
payment records, or router compromise, do not post the details publicly. Follow
the security reporting process in `SECURITY.md`.

## Enforcement

Vexel Innovations maintainers may edit, hide, or remove content that violates
this code. Repeat or severe violations may lead to revoked access to the
repository or project channels.
