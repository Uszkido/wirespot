# WireSpot Code Of Conduct

WireSpot is a professional Vexel Innovations project for MikroTik RouterOS
hotspot operators, network engineers, Android developers, and support partners.
Every contributor, tester, maintainer, and collaborator is expected to keep the
project respectful, secure, practical, and original.

## Our Standards

Expected behavior:

- Communicate with respect, patience, and technical clarity.
- Keep discussions focused on WireSpot, Flutter, Android, RouterOS, WireGuard,
  printing, reporting, licensing, and hotspot operator workflows.
- Share reproducible steps when reporting issues.
- Protect router credentials, VPN keys, customer records, sales records, and
  private business information.
- Credit original work and avoid copying proprietary applications, source code,
  design assets, templates, or private documentation.
- Prefer secure deployment guidance, especially WireGuard or trusted local LAN
  access instead of exposed public RouterOS API ports.

Unacceptable behavior:

- Harassment, insults, intimidation, threats, or discriminatory language.
- Publishing real RouterOS passwords, WireGuard private keys, Android signing
  keys, billing secrets, customer data, or production screenshots with secrets.
- Submitting copied code from Mikhmon, WireNex, or any proprietary application.
- Encouraging insecure deployment patterns that place operators or customers at
  avoidable risk.
- Spam, unrelated promotion, abusive review comments, or disruptive behavior.

## Security-Sensitive Discussions

Do not post sensitive security details publicly. If a report involves
authentication, credentials, VPN configuration, payment/licensing records,
RouterOS compromise, private keys, or customer data, follow the private
reporting process in [SECURITY.md](SECURITY.md).

## Enforcement

Vexel Innovations maintainers may edit, hide, remove, or reject content that
violates this code. Maintainers may also restrict repository or project-channel
access for repeated, severe, or security-risking behavior.

When enforcement is needed, maintainers should act proportionally and document
the reason internally.

## Scope

This Code of Conduct applies to:

- GitHub issues, pull requests, discussions, and reviews.
- Project documentation and commit messages.
- WireSpot support channels operated by Vexel Innovations.
- Any collaboration space used for WireSpot development, testing, or release.
